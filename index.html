Hybrid and multi-cloud

Ploigos enables deployment and management of Kubernetes and Container workloads in a real hybrid approach, having single clusters deployed in heterogenous infrastructures, migrating live clusters and more

Enterprise ready

Out of the box, Ploigos Kubernetes includes enterprise features like network security, policies, fabric, service mesh, disaster recovery, AD/LDAP/Authorization integration, observability, reporting and more

Internal Developer Platform

With the push of a button, each cluster has internal identity providers to provide auth backend as a service, database and message queues as a service, FaaS, and more. Blueprints for containerization enable application modernization and cloud transition.

Platform Features

Hybrid and multi cloud

Ploigos enables true hybrid-cloud strategies, giving you the ability to deploy single clusters across different cloud providers and on-premise servers.

Authentication and authorization

Kidman is our service responsible for user federation, identity brokering and providing identity as a service for cluster services and applications

Advanced security

Ploigos provides hardened Kubernnetes and CaaS infrastructure, with built-in features for secrets management, AMI and OCI image hardening and scanning and more.

Advanced cloud-native features

Our Kubernetes distribution comes out of the box with advanced network setups with custom CNI plugins for cross-cluster communication, network fabric, service mesh, multiple ingress options and more

Compliance & reporting

With a click of a button, clusters and applications can automatically report user access, network policies and more, to ease compliance reporting like SOC2, PCI, FedRAMP etc

Everything-as-a-Service

With Ploigos you get a single, easy to manage setup, which provides multiple deployment options. Connect applications from Kubernets, CaaS, PaaS, FaaS, and more.
